=============================================

OBJECTIVE:
Optimize visible_indices calculations to only calculate visible range, not full list.

   - Takes scroll_state, selected_position, len, viewport_lines, header_lines, footer_lines
   - Returns (window_start, window_len) tuple for the visible range

3. Current flow analysis for repo_branch_lines() at line 7568:

   - Then uses detached_list_window() to compute window_start/window_len
   - Then uses .skip(window_start).take(window_len) to limit iteration
   
   The iteration is already efficient - skip/take happens during the loop,
   not during index calculation. The visible indices filtering is necessary
   because branch_matches_filter() must be called for each branch to know
   which ones match the filter query.

4. Caching is already implemented:
   - repo_scroll_state() at line 682 retrieves state from self.repo_scroll HashMap
   - This state is cached between frames in the AppState struct
   - The scroll offset persists across frames until user scrolls
